Run the app on your Claude subscription via the Agent SDK

Re-scoped from the raw API. Replace Pydantic AI + DeepSeek in the plan-agent service with the Claude Agent SDK (claude-agent-sdk), authenticated by your Claude subscription (CLAUDE_CODE_OAUTH_TOKEN from claude setup-token) — no paid API key. Structured JSON stays intact via the SDK's output_format json-schema.

branchagent-sdk authsubscription token sdkclaude-agent-sdk modelclaude-opus-4-8 scope1 service · 3 call sites
Why this replaces the API-key plan. The raw anthropic SDK (messages.parse) can only bill a paid API key — a subscription can't power it. The Agent SDK wraps the Claude Code CLI, so it accepts a subscription OAuth token. Trade-off: you're now bound by subscription rate limits (Pro is tight — Max recommended), and the app is effectively single-tenant per token (§7). This is the right shape only because the app runs locally on your PC.
00

One-time — mint your subscription token

Do this once on the machine that runs the app. The token is a 1-year, inference-only OAuth credential tied to your Pro/Max plan.

claude setup-token # OAuth via your Pro/Max login → prints a 1-year token export CLAUDE_CODE_OAUTH_TOKEN=<token> # paste into app/.env (gitignored) unset ANTHROPIC_API_KEY # CRITICAL: it outranks the token & bills API credits
Credential precedence (from Claude Code docs): cloud provider → ANTHROPIC_AUTH_TOKEN → ANTHROPIC_API_KEY → apiKeyHelper → CLAUDE_CODE_OAUTH_TOKEN → /login subscription. The token is near the bottom, so any stray API key wins — the app would silently bill credits instead of your subscription. The plan's health check surfaces which is active.
01

Goal & what changes

Collapse the provider-pluggable Pydantic AI stack to a single path: the Claude Agent SDK, run tool-free and single-turn as a structured-extraction call. The geometry contract, validated op pipeline, echo fallback, backend, and frontend are untouched — only the three LLM call sites and their auth change.

In scope

  • Swap Pydantic AI Agent → claude_agent_sdk.query() at all 3 sites
  • Structured JSON via output_format={json_schema} → ResultMessage.structured_output
  • Vision image passed as a base64 content block (streaming prompt)
  • Remove DeepSeek/OpenAI + pydantic-ai-slim; add claude-agent-sdk
  • Auth on CLAUDE_CODE_OAUTH_TOKEN; clear error when absent
  • Tool-free, max_turns=1 — behaves like one structured LLM call
  • Update env, compose, Dockerfile, health, docs, spec D5

Explicitly untouched

  • plan-core geometry, ops, validator, compliance
  • backend-api (still calls agent over HTTP), Postgres, seed
  • frontend — never talked to a model directly
  • The validator bounce-back retry loop & prompt wording (parity)
  • AGENT_MODE=echo LLM-free path
  • Tavily search (only the extraction step becomes Claude)

Proposed approach

One shared module — llm.py — owns the SDK call, the auth guard, model config, and a single parse_structured() helper that runs query() tool-free / single-turn and returns a validated Pydantic model from structured_output. The three call sites stay thin; the domain bounce-back loop is untouched.

07

Risks & mitigations

RiskSevWhyMitigation
Op-union schema rejected by output_format high OpsPlan.ops is list[Op], a discriminated union. model_json_schema() emits $defs/anyOf; json-schema validation may reject recursion/refs. Probe first (phase 1). If rejected: flatten the schema, or drop output_format and prompt for JSON in the system prompt + parse the final text block (the same models still validate it).
Subscription rate limits high Ops loop (≤3 attempts) + vision + comps per review iteration draw on your plan quota. Pro throttles quickly; a busy session can stall. Use Max for real use. Keep comps/ops on Opus only if quota allows; env-override to Sonnet to stretch limits. Surface 429/limit errors from the SDK as clear 502s.
ANTHROPIC_API_KEY precedence trap med If any API key is set in the env, it silently overrides the subscription token and bills API credits. Ensure it's unset in shell, .env, and the container. /health reports which credential is active; add a startup warning if a key is present.
Bundled CLI in the Docker image med pip install claude-agent-sdk bundles the CLI, but the binary must run in the slim-python linux image (glibc/arch). Verify at build (phase 5). Fallbacks: curl … install.sh in the Dockerfile, or set ClaudeAgentOptions(cli_path=…).
Token expiry / terms low Token lasts ~1 year, inference-only. Automated subscription use has terms limits. Re-run claude setup-token to renew; health surfaces expiry failures. Personal local use is within the token's intent — flag anything beyond.
Latency / subprocess low query() spawns the CLI subprocess per call. Backend /apply already uses a 600s timeout. Small structured outputs → acceptable; a pooled ClaudeSDKClient is a later optimisation.

09

Non-goals (this change)

  • Multi-tenant per-user subscriptions. Each operator supplies their own CLAUDE_CODE_OAUTH_TOKEN; the app doesn't broker distinct users' subscriptions in one process.
  • The raw anthropic API path. Superseded — it can't use a subscription. (Kept as an env-swap fallback only if you later want pay-per-token.)
  • Streaming to the frontend. Outputs are small structured objects.
  • Pooled/session-reuse SDK client. A later latency optimisation; v1 uses a fresh query() per request.
  • Claude Code's built-in tools (Read/Bash/Edit). Deliberately disabled — this is a structured-extraction call, not a coding agent.
